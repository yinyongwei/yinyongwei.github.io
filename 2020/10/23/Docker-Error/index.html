记录一次Docker 启动失败的问题 inotify_add_watch(7, /dev/dm-1, 10) failed: No such file or directory

使用了阿里云的一键安装脚本 curl -fsSL https://get.docker.com | bash -s docker --mirror Aliyun 在 移动云新服务上进行了执行安装， 然后通过命令，进行启动systemctl start docekr

出现报错

它提示我，使用 systemctl status docker.service 和 journalctl -xe 命令进行查看；

采用 systemctl status docker.service

采用 journalctl -xe

红色框框标记出，提示 inotify_add_watch(7, /dev/dm-1, 10) failed: No such file or directory

原因:

xfsprogs版本过低

解决方法:

yum update xfsprogs

重启docker:

systemctl daemon-reload

systemctl restart docker
